# アーキテクチャ設計書

## システム構成

```
┌─────────────────────────────────────────────────────────┐
│               音声入力源                                   │
│  マイク / ステレオミキサー（Zoom・YouTube など）           │
└─────────────────────┬───────────────────────────────────┘
                      │ PCM 16kHz float32
                      ▼
┌─────────────────────────────────────────────────────────┐
│               sounddevice (PortAudio)                    │
│  InputStream（コールバックスレッド）                       │
│  blocksize = 4800 サンプル（0.3秒）                       │
└─────────────────────┬───────────────────────────────────┘
                      │ numpy 配列（チャンク単位）
                      ▼
┌─────────────────────────────────────────────────────────┐
│               AudioProcessor (VAD)                       │
│  エネルギーベース無音検出（RMS閾値 0.005）                  │
│  無音 1.5 秒継続 → 発話バッファを Queue に push            │
│  最大 30 秒で強制フラッシュ                               │
└─────────────────────┬───────────────────────────────────┘
                      │ 発話単位の numpy 配列
                      ▼
┌─────────────────────────────────────────────────────────┐
│            faster-whisper（完全ローカル）                  │
│  モデル: tiny / base / small / medium                    │
│  量子化: int8（CPU最適化）                                │
│  言語固定: en / vad_filter=True / IT専門用語プロンプト     │
│  音声データは PC 外に一切出ない                           │
└─────────────────────┬───────────────────────────────────┘
                      │ 英語テキスト
                      ▼
┌─────────────────────────────────────────────────────────┐
│          deep-translator / Google Translate              │
│  HTTPS で Google サーバーに送信                           │
│  ※ 送信されるのはテキストのみ（音声は送らない）             │
└─────────────────────┬───────────────────────────────────┘
                      │ 日本語テキスト
                      ▼
┌─────────────────────────────────────────────────────────┐
│               tkinter GUI                                │
│  左ペイン: 英語原文（タイムスタンプ付き）                   │
│  右ペイン: 日本語翻訳（タイムスタンプ付き）                 │
└─────────────────────────────────────────────────────────┘
```

## スレッド設計

```
メインスレッド（tkinter イベントループ）
  │
  ├─ sounddevice コールバックスレッド（OS管理）
  │    → チャンクを _audio_q に put
  │
  ├─ _feed_loop スレッド（daemon）
  │    → _audio_q から取り出し、VAD に feed
  │    → 発話バッファを vad.ready に push
  │
  └─ _transcribe_loop スレッド（daemon）
       → vad.ready から取り出し
       → faster-whisper で文字起こし
       → Google翻訳 → _result_q に put

メインスレッドの after(100ms) ポーリング
  → _result_q から取り出してGUIに表示
```

全スレッドは `daemon=True` のため、アプリ終了時に自動終了する。

## 主要な設計判断

### 1. エネルギーベース VAD を自前実装した理由

Silero VAD（ML）や WebRTC VAD（C拡張）を使わず、RMS計算のみで実装している。

- 依存ライブラリを増やさない（インストール摩擦を下げる）
- RMS閾値はIT会議の音量レンジで十分に機能する
- 遅延がほぼゼロ（ML推論のオーバーヘッドなし）

### 2. キュー経由の非同期パイプライン

音声取得・VAD処理・文字起こしをスレッド間 Queue で繋ぐことで、各処理が独立して進む。文字起こしに数秒かかっても音声の取りこぼしが発生しない。

### 3. faster-whisper + int8 量子化

オリジナル Whisper より CPU 使用率を2〜4倍改善するため、CTranslate2 の int8 量子化を使用。GPU なしの一般 PC でも実用速度（smallモデルで 1〜3 秒/発話）を実現している。

### 4. IT専門用語プロンプト

Whisper の `initial_prompt` に IT 用語集を渡すことで、「API」「Kubernetes」「CI/CD」などの認識精度を高めている。Zoom でのエンジニアリング会議を主なユースケースとして想定した設計。

### 5. モデル選択をランタイムに変更可能

起動後にドロップダウンからモデルサイズを変更できる。Zoom 通話中はCPU節約のため `base` を選び、録画の後処理には精度優先で `medium` を使うなど、用途に応じた使い分けを想定している。
