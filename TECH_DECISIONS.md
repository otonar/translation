# 技術選定理由書

## 1. 音声認識（STT）: faster-whisper

### 選定理由

| 比較対象 | faster-whisper | OpenAI Whisper（本家） | Google Cloud STT | Azure Speech |
|---------|---------------|----------------------|-----------------|-------------|
| 実行場所 | **ローカル** | ローカル | クラウド | クラウド |
| 音声の外部送信 | **なし** | なし | **あり** | **あり** |
| 費用 | **無料** | 無料 | 従量課金 | 従量課金 |
| CPU速度 | **2〜4倍高速** | 基準 | — | — |
| APIキー | **不要** | 不要 | 必要 | 必要 |
| IT専門用語プロンプト | **◎** | ◎ | △ | △ |
| モデル選択 | **tiny〜large** | tiny〜large | 自動 | 自動 |

**採用決定の理由:**

- SYSTRAN 製の CTranslate2 バックエンドにより、オリジナル Whisper より CPU 推論が2〜4倍高速
- int8 量子化で GPU なしの一般 PC でも実用速度を達成
- 音声データが一切外部に出ないため、社内会議・NDA対象コンテンツにも使いやすい
- `initial_prompt` パラメータで IT 専門用語の認識精度をチューニングできる

### Vosk を選ばなかった理由

Vosk もオフライン STT だが、Whisper 系と比較して認識精度が低い（特に英語の技術用語）。また `initial_prompt` のような推論時のカスタマイズができない。

### Google Cloud STT・Azure Speech を選ばなかった理由

クラウド STT は音声データをサーバーに送信する。Zoom 会議の内容が外部に送られることに抵抗を感じるユーザーが多いと判断した。また、APIキーの取得・管理が必要で導入摩擦が高い。

## 2. 翻訳: deep-translator（Google Translate）

### 選定理由

| 比較対象 | Google Translate（無料版） | DeepL | OpenAI/Claude API | opus-mt（ローカル） |
|---------|--------------------------|-------|-------------------|-------------------|
| 費用 | **無料** | 無料枠あり・有料版 | **従量課金** | **無料** |
| APIキー | **不要** | 必要 | 必要 | 不要 |
| 翻訳品質 | ◎ | **◎（自然な日本語）** | **◎（最高品質）** | △ |
| インターネット | 必要 | 必要 | 必要 | **不要** |
| 導入の簡単さ | **◎** | ○ | △ | △（モデル大） |

**採用決定の理由:**

- APIキー不要で `pip install` だけで使える（セットアップ摩擦ゼロ）
- IT技術文書の翻訳品質は実用上十分
- 「Google翻訳のサイトに貼り付ける」行為と同等のため、ユーザーがリスクを直感的に理解しやすい
- deep-translator ライブラリが複数の翻訳バックエンドを同一インターフェースで切り替えられる設計のため、将来 DeepL に移行しやすい

### ローカル翻訳モデルを選ばなかった理由

Helsinki-NLP/opus-mt-en-jap などのローカルモデルは追加で数百MB〜数GBのダウンロードが必要。また翻訳品質が Google 翻訳より劣り、推論時間も長い。Whisper の STT がすでにローカルで動いているため、翻訳はクラウドを使うトレードオフを選んだ。

## 3. 音声入力: sounddevice

### 選定理由

| 比較対象 | sounddevice | PyAudio | pyaudiowpatch |
|---------|-------------|---------|---------------|
| PortAudioバイナリ | **同梱** | 別途必要 | 同梱 |
| ステレオミキサー対応 | **◎** | ◎ | **◎（Windows専用強化）** |
| メンテナンス状況 | **活発** | 停滞気味 | 限定的 |
| numpy統合 | **ネイティブ** | 手動変換 | 手動変換 |
| コールバック形式 | **◎** | ◎ | ◎ |

**採用決定の理由:**

- 入力データが直接 `numpy.ndarray` で渡されるため、Whisper への受け渡しに変換不要
- PortAudio のバイナリが同梱されているため、`pip install sounddevice` だけで動く
- sounddevice のコールバック形式は `blocksize` で量子化されたチャンクが来るため、VAD 実装との相性が良い

## 4. VAD（無音検出）: 自前エネルギーベース実装

### 選定理由

| 比較対象 | 自前（RMS） | Silero VAD | WebRTC VAD |
|---------|------------|------------|------------|
| 追加依存 | **なし** | torch（重い） | C拡張必要 |
| 精度 | △〜◎（閾値次第） | **◎（ML）** | ○ |
| レイテンシ | **ほぼゼロ** | 数ms | 数ms |
| 実装の透明性 | **高い** | 低い | 中 |

**採用決定の理由:**

- PyTorch（約2GB）を追加インストールせずに済む
- RMS計算で十分に実用的な VAD が実現できる（IT会議の音量では誤検出が少ない）
- ソースコード100行以内でロジックが完結しており、閾値や継続時間のチューニングが簡単

### トレードオフ

背景ノイズが多い環境（カフェ・空調音の強い部屋）では誤検出が増える可能性がある。その場合は `SILENCE_THRESHOLD` を引き上げることで対処できる。

## 5. GUI: tkinter

### 選定理由

| 比較対象 | tkinter | PyQt6 | wxPython | Dear PyGui |
|---------|---------|-------|----------|------------|
| Python同梱 | **◎（標準ライブラリ）** | × | × | × |
| 追加インストール | **不要** | 必要 | 必要 | 必要 |
| Windows対応 | ◎ | ◎ | ◎ | ◎ |
| ダークテーマ実装 | 手動 | 標準 | 手動 | **標準** |
| 学習コスト | **低** | 高 | 高 | 中 |

**採用決定の理由:**

- Python 標準ライブラリのため `pip install` 不要。`requirements.txt` を最小限に保てる
- 2ペイン分割・スクロールテキスト・ステータスバーという要件に対して tkinter で十分
- ttk スタイルと手動カラー設定で Catppuccin ベースのダークテーマを実現

### PyQt を選ばなかった理由

PyQt6 はより高機能な GUI を作れるが、このアプリに必要な機能は tkinter で完全に満たせる。`pip install PyQt6` という追加手順はユーザーの導入摩擦を増やすだけで、価値を生まない。
